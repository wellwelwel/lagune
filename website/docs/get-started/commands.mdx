# Commands

Once **Blue Spec** is set up in your project, your **AI** agent unlocks a set of slash commands.

## The Blue Team flow

These five run in order. Each builds on the previous, so following the list top to bottom is all it takes.

| #   | Command                                         | What it does for you                                                           |
| --- | ----------------------------------------------- | ------------------------------------------------------------------------------ |
| 1   | [**/bluespec.charter**](/docs/commands/charter) | Sets your project's security rules, proposed for you or shaped by what you say |
| 2   | [**/bluespec.detect**](/docs/commands/detect)   | Reads your code and maps what your system does and where the risks are         |
| 3   | [**/bluespec.plan**](/docs/commands/plan)       | Turns what detect found into a defense plan, with a fix for each finding       |
| 4   | [**/bluespec.harden**](/docs/commands/harden)   | Applies the plan's fixes to your code, safely and one at a time                |
| 5   | [**/bluespec.verify**](/docs/commands/verify)   | Proves each applied fix holds and closes out the ones that do                  |

> Each command builds on the previous, so following the list top to bottom is all it takes.

#### Special commands

| Command                                               | What it does                                                                               |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| [**/bluespec.specialize**](/docs/commands/specialize) | Specializes **Blue Spec** in a new security _sub_-skill from articles, exploits, or topics |
| [**/bluespec.skills**](/docs/commands/skills)         | Loads an on-demand security _sub_-skill                                                    |
| [**/bluespec.repair**](/docs/commands/repair)         | Repairs **Blue Spec**'s internal tracking                                                  |
| [**/bluespec.list**](/docs/commands/list)             | Lists all finding **Blue Spec** is tracking, by name                                       |
